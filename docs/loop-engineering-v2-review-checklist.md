# Loop Engineering v2 Review Checklist

- [ ] Discovery and Delivery are clearly separated.
- [ ] Every important loop has red-capable evidence.
- [ ] Verifier owner, version, blind spots, and review cadence are explicit where needed.
- [ ] Maker cannot silently redefine success criteria.
- [ ] Acceptance reports requested and achieved independence.
- [ ] Human-only and irreversible decisions are visible.
- [ ] `CHECKPOINT.md` preserves rationale without replaying the full run log.
- [ ] Multi-loop repositories can use stable named loop directories.
- [ ] `clarify`, `dev`, and `acceptance` share the same context model.
- [ ] New references are checked by `skills-doctor.sh`.
